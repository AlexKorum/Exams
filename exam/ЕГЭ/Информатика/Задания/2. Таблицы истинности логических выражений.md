# Описание
В этом типе заданий требуется работа с таблицей истинности логической функции, в которой переменные (например, *w, x, y, z*) не привязаны к конкретным столбцам. Предоставляется фрагмент таблицы — несколько строк с известными значениями функции и переменных, но без указания, какой столбец соответствует какой переменной. А также предоставляется сама логическая функция.

Цель — установить соответствие между переменными и столбцами таблицы, опираясь на логическую структуру функции и значения в таблице. Ответ записывается в виде последовательности букв, обозначающих переменные, в порядке следования столбцов слева направо — без пробелов и разделителей.

---
# Принцип решения
## Общая логика подхода

Такие задачи требуют определить соответствие между переменными логической функции и столбцами таблицы истинности, когда это соответствие не задано явно. Известны лишь фрагмент таблицы — несколько строк с частично заполненными значениями переменных и результатом функции.

Решение строится на **систематическом переборе всех возможных вариантов**, поскольку прямое логическое выведение часто невозможно из-за недостатка данных.

### Основные этапы решения:

1. **Перебор неизвестных значений**  
   В таблице могут присутствовать пустые ячейки — неизвестные значения переменных. Все возможные комбинации этих значений перебираются полным перебором.

2. **Проверка уникальности строк**  
   Каждая строка таблицы должна быть уникальной — это обязательное условие для корректной таблицы истинности. Проверка осуществляется путём сравнения количества строк с количеством уникальных строк.

3. **Перебор всех возможных соответствий переменных столбцам**  
   Поскольку неизвестно, какая переменная соответствует какому столбцу, перебираются все возможные перестановки переменных по столбцам. Это обеспечивает полный охват всех гипотез.

4. **Проверка соответствия функции**  
   Для каждой комбинации значений и каждого варианта сопоставления переменных столбцам вычисляется значение логической функции и сравнивается с указанным в таблице. Если все строки совпадают — найдено верное решение.

5. **Вывод результата**  
   Как только найдено соответствие, удовлетворяющее всем условиям, оно фиксируется в виде последовательности букв, соответствующих столбцам слева направо.

## Почему такой подход эффективен?

В условиях недостатка данных и отсутствия явной логической связи между столбцами и переменными — **перебор является надёжным методом**. Он гарантирует, что будет найдено единственное верное решение, если оно существует.

# Шаблон кода
```python
from itertools import product, permutations  
  
  
def f(x, y, z, w):  
    return (w == z) or not (y <= w) or not x  
  
  
for x1, x2, x3, x4, x5 in product([0, 1], repeat=5):  
    t = (  
        (x1, 0, 1, 0, 0),  
        (x2, 1, 1, x3, 0),  
        (0, x4, x5, 0, 0)  
    )  
    if len(t) == len(set(t)):  
        for p in permutations('xyzw', r=4):  
            if all(f(**dict(zip(p, m))) == m[-1] for m in t):  
                print(*p, sep='')
```
