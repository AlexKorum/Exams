# Описание

В этом типе заданий требуется выполнить арифметические операции с числами, записанными в различных системах счисления, и проанализировать их представление в заданной системе счисления. Часто требуется найти количество определённых цифр в записи числа, определить основание системы счисления, при котором выполняется равенство, или найти значение переменной, при котором в записи числа выполняются заданные условия (например, количество нулей, кратность числу и т.п.).

Задачи могут включать:
- вычисление значения сложного арифметического выражения;
- перевод числа из десятичной системы в заданную систему счисления;
- подсчёт количества определённых цифр в записи числа;
- нахождение минимального или максимального основания системы счисления, при котором выполняется равенство;
- определение значения переменной, при котором выполняются условия (например, количество нулей, кратность и т.п.).

# Принцип решения

## Общая логика подхода

Решение таких задач строится на последовательном выполнении следующих шагов:

1. **Вычисление значения выражения в десятичной системе счисления**  
   Все операнды и промежуточные результаты вычисляются в десятичной системе. Это позволяет избежать ошибок, связанных с арифметикой в недесятичных системах.

2. **Перевод числа в заданную систему счисления**  
   Используется функция `dec_to_base(n, b)`, которая переводит число `n` из десятичной системы в систему с основанием `b`.  
   Важно: при переводе необходимо учитывать, что цифры от 10 и выше обозначаются буквами (A, B, C, ...).

3. **Анализ записи числа в заданной системе счисления**  
   После перевода анализируется строковое представление числа:
   - подсчёт количества определённых цифр (например, цифр "1", "4", "5", "2" и т.п.);
   - проверка наличия нулей или других условий;
   - определение длины записи, позиций цифр и т.п.

4. **Поиск основания системы счисления или значения переменной**  
   В задачах, где требуется найти основание системы счисления или значение переменной, используется перебор:
   - для основания — перебор значений, начиная с минимально возможного (обычно на 1 больше максимальной цифры в записи чисел);
   - для переменной — перебор значений в заданном диапазоне (например, от 1 до 27000, 500000 и т.п.);
   - для каждого значения проверяется выполнение условия (равенство, кратность, количество нулей и т.п.).

5. **Проверка условия и вывод результата**  
   Как только найдено значение, удовлетворяющее условию, оно фиксируется в ответе. В задачах с перебором часто требуется найти минимальное или максимальное значение, удовлетворяющее условию.

## Почему такой подход эффективен?

- Вычисления в десятичной системе счисления исключают ошибки, связанные с арифметикой в недесятичных системах.
- Перебор позволяет найти точное решение, даже если аналитическое решение сложно или невозможно.
- Функции перевода и анализа записи числа являются универсальными и применимы ко всем задачам данного типа.

# Шаблон кода

```python
from string import digits, ascii_uppercase
from itertools import product

def dec_to_base(n: int, b: int) -> str:
    if n == 0:
        return '0'
    alphabet = digits + ascii_uppercase
    result = []
    while n > 0:
        result.append(alphabet[n % b])
        n //= b
    return ''.join(reversed(result))

def base_to_dec(n: str, b: int) -> int:
    return sum(int(el, 36) * (b ** i) for i, el in enumerate(reversed(n)))

# Пример решения задачи с подсчётом цифр
def solve_count_digits():
    # Вычисление значения выражения
    n = (16**350 * (15 * 3 - 29)**424 + 1007) // 63
    # Перевод в систему с основанием 4
    s = dec_to_base(n, 4)
    # Подсчёт цифр '1'
    count = s.count('1')
    return count

# Пример решения задачи с поиском основания системы счисления
def solve_find_base():
    # Перебор оснований, начиная с минимального
    for p in range(10, 500001):  # Минимальное основание — 10, максимальное — 500000
        try:
            # Вычисление левой и правой частей уравнения в системе с основанием p
            left = base_to_dec('29A1', p) + base_to_dec('4771', p) + base_to_dec('12A', p)
            right = base_to_dec('100000', p) + 210
            if left == right:
                return p
        except:
            continue
    return None

# Пример решения задачи с поиском значения переменной
def solve_find_x():
    # Перебор значений x в заданном диапазоне
    for x in range(1, 27001):
        # Вычисление значения выражения
        n = 3 * 27**9 + 2 * 27**6 + 27**3 - x
        # Перевод в 27-ричную систему
        s = dec_to_base(n, 27)
        # Проверка количества нулей
        if s.count('0') == 6:
            return x
    return None
```

# Примеры

**Пример 1.**  
Значение выражения `(16^350 * (15 * 3 - 29)^424 + 1007) / 63` записали в системе счисления с основанием 4. Сколько цифр 1 в этой записи?  
→ Решение: вычислить значение выражения, перевести в систему с основанием 4, подсчитать количество цифр '1'.

**Пример 2.**  
Операнды равенства `29A1_p + 4771_p + 12A_p = 100000_p + 210` записаны в системе счисления с основанием p. Определите наименьшее возможное основание p.  
→ Решение: перебрать основания p, начиная с 10, проверить равенство для каждого p, вернуть первое подходящее.

**Пример 3.**  
Значение выражения `3 * 27^9 + 2 * 27^6 + 27^3 - x` записали в 27-ричной системе счисления. Определите наименьшее значение x, при котором в записи числа ровно 6 нулей.  
→ Решение: перебрать x от 1 до 27000, для каждого вычислить выражение, перевести в 27-ричную систему, проверить количество нулей, вернуть первое подходящее x.

**Пример 4.**  
Значение выражения `500 * (4 * 6 + 1)^5 + 7 / 128` записали в системе счисления с основанием 5. Сколько цифр 4 в этой записи?  
→ Решение: вычислить значение выражения, перевести в систему с основанием 5, подсчитать количество цифр '4'.

**Пример 5.**  
Дано арифметическое выражение: `SLADOST_36 + GADOST_10x = HALLOWEEN_50-x - 166729861760449_10`. Определите наименьшее возможное значение x, при котором выполняется равенство.  
→ Решение: перебрать x, вычислить левую и правую части в десятичной системе, проверить равенство, вернуть первое подходящее x.
